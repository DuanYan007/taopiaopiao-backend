```
时间轴:
  │
  │  预约期 (T-7天 ~ T-1小时)

  │  └─ 限制预约人数/资格

  │  └─ 构建倒计时
  │
  │  售卖期 (T时刻开始)
  │  ├─ 正式选座下单
  │  ├─ seat-lock-service + RocketMQ
  │  └─ 高并发抢座

  ▼
```

## 三、数据结构设计

### 3.1 Redis 数据结构

| Key | 类型 | 说明 | TTL |
|-----|------|------|-----|
| `appointment:session:{sessionId}:users` | Set | 预约用户集合 | 预约结束+1天 |
| `appointment:session:{sessionId}:quota` | String | 预约名额 | 预约结束+1天 |
| `appointment:user:{userId}:sessions` | Set | 用户已预约场次 | 永久 |
| `preheat:session:{sessionId}:status` | String | 预热状态: 0-未预热, 1-预热中, 2-已预热 | 24小时 |
| `seats:{sessionId}:{seatId}` | String | 座位状态: 0-可售, 1-锁定, 2-已售 | 锁定时长 |
| `seats:{sessionId}:sold` | Set | 已售座位集合 | 永久 |
| `seats:{sessionId}:available` | Set | 可售座位集合 | 永久 |
| `session:{sessionId}:status` | String | 场次状态: RESERVED(预约), PREHEAT(预热), ONSALE(开售) | 永久 |

## 四、详细流程设计

### 4.1 阶段一：预约期 (T-7天 ~ T-1小时)

#### 时间设置
```
appointment_start_time = T - 7天
appointment_end_time   = T - 1小时
sale_start_time        = T
```

#### 4.1.1 预约初始化 (后台定时任务)

```java
// 服务: session-service 或 独立的 scheduler-service
// 触发时间: appointment_start_time 到期

public void initAppointment(Long sessionId) {
    // 1. 更新数据库
    session.setSaleStage("RESERVED");
    sessionMapper.updateById(session);

    // 2. Redis 初始化预约集合
    redis.del("appointment:session:" + sessionId + ":users");
    redis.set("appointment:session:" + sessionId + ":quota", quota);

    // 3. 发送MQ消息通知
    mq.send(APPOINTMENT_INIT_MSG);
}
```

#### 4.1.2 用户预约

```
前端请求
    │
    ▼
OpenResty 网关 (Lua校验)
    │
    ├─ 检查场次是否在预约期内
    │
    ├─ 检查用户是否已预约 (SISMEMBER)
    │   redis.sismember("appointment:session:{sessionId}:users", userId)
    │
    ├─ 检查预约名额是否已满
    │   quota = redis.get("appointment:session:{sessionId}:quota")
    │   count = redis.scard("appointment:session:{sessionId}:users")
    │
    ├─ 添加用户到预约集合 (SADD)
    │   redis.sadd("appointment:session:{sessionId}:users", userId)
    │
    ├─ 记录用户预约 (SADD)
    │   redis.sadd("appointment:user:{userId}:sessions", sessionId)
    │
    └─ 限流: 每用户每场次只能预约1次
```

**OpenResty Lua 脚本示例**:
```lua
-- appointment_check.lua
local redis = require "resty.redis"
local red = redis:new()

local sessionId = ngx.var.session_id
local userId = ngx.var.user_id

-- 检查预约状态
local stage = red:get("session:" .. sessionId .. ":status")
if stage ~= "RESERVED" then
    return ngx.exit(403) -- 非预约期
end

-- 检查是否已预约
if red:sismember("appointment:session:" .. sessionId .. ":users", userId) == 1 then
    return ngx.exit(400) -- 已预约
end

-- 检查名额
local quota = tonumber(red:get("appointment:session:" .. sessionId .. ":quota"))
local count = tonumber(red:scard("appointment:session:" .. sessionId .. ":users"))
if count >= quota then
    return ngx.exit(409) -- 名额已满
end

-- 添加预约
red:sadd("appointment:session:" .. sessionId .. ":users", userId)
red:sadd("appointment:user:" .. userId .. ":sessions", sessionId)
```

### 4.2 阶段二：预热期 (T-1小时 ~ T-5分钟)

#### 4.2.1 预热触发

```java
// 定时任务: appointment_end_time 到期触发

public void preheatSession(Long sessionId) {
    // 1. 更新场次状态
    redis.set("session:" + sessionId + ":status", "PREHEAT");
    sessionMapper.updateSaleStage(sessionId, "PREHEAT");

    // 2. 从数据库加载座位数据
    List<Seat> seats = seatMapper.selectList(
        new QueryWrapper<Seat>()
            .eq("session_id", sessionId)
            .eq("status", "available")
    );

    // 3. 批量写入 Redis
    Pipeline pipeline = redis.pipelined();
    for (Seat seat : seats) {
        // 初始化座位状态为可售 (0)
        pipeline.setex("seats:" + sessionId + ":" + seat.getId(), 3600, "0");
        // 加入可售集合
        pipeline.sadd("seats:" + sessionId + ":available", seat.getId());
    }
    pipeline.sync();

    // 4. 锁定预约用户集合，防止新增
    redis.expire("appointment:session:" + sessionId + ":users", 3600);

    // 5. 更新预热状态
    redis.set("preheat:session:" + sessionId + ":status", "2");
}
```

#### 4.3.2 资格校验 (OpenResty)

```
用户请求选座/下单
    │
    ▼
OpenResty 网关
    │
    ├─ 检查场次状态是否为 ONSALE
    │
    ├─ 检查用户是否有预约资格 (SISMEMBER)
    │   redis.sismember("appointment:session:{sessionId}:users", userId)
    │

    │
    └─ 如果有预约资格，放行到 seat-lock-service
```

**OpenResty Lua 脚本示例**:
```lua

    return ngx.exit(403) -- 未开售
```

#### 4.3.3 选座下单 (seat-lock-service)

```
POST /seats/lock

请求:
{
    "sessionId": 1,
    "userId": 100,
    "seatIds": ["101", "102"]
}

处理流程:
    │
    ├─ Redis Lua: 检查座位状态
    │   • 座位是否存在
    │   • 座位是否可售 (status=0)
    │   • 是否被锁定 (status=1)
    │
    ├─ Redis Lua: 锁定座位
    │   • seats:{sessionId}:{seatId} = 1
    │   • 设置过期时间 (15分钟)
    │
    ├─ 生成订单号
    │
    ├─ 立即返回
    │
    └─ 发送MQ消息: SEAT_LOCKED
```

```
POST /seats/confirm

请求:

处理流程:
```

## 七、注意事项

### 7.1 并发控制
- **预约**: Redis Set 天然去重，配合 OpenResty 限流
- **选座**: Redis Lua 脚本保证原子性
- **支付**: 分布式锁 + 消息幂等

### 7.2 数据一致性
- Redis 为准实时数据，DB 为最终数据
- 定时对账任务确保 Redis 与 DB 一致
- 消息丢失补偿机制

### 7.3 降级方案
- **Redis 故障**: 降级直接查询 DB（性能下降）
- **MQ 故障**: 降级同步调用 DB 服务
- **OpenResty 故障**: 旁路直接调用后端

### 7.4 监控指标
- 预约完成率
- 预热成功率
- 选座成功率
- 支付成功率
- Redis 命中率
- 消息堆积量
